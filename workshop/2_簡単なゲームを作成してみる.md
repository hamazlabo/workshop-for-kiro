# 簡単なゲームを作成してみる

## このセクションの目的

Kiro CLI を使って、簡単なゲームを作成してみる。
これによって、AI との対話や、使い方のイメージを付ける。

## 2.0. 事前準備

ターミナルを開いて、カレントディレクトリを確認する。

```bash
pwq
# 次のように出力される想定: /home/ubuntu/workshop-for-kiro
```

異なるディレクトリが表示された場合に実施:

```bash
cd ~/workshop-for-kiro
```

## 2.1. 手順

### 2.1.1. Kiro を起動

```bash
kiro-cli
```

### 2.1.2. エージェントに指示する

```txt
簡単なテトリスゲームを作成して下さい。
```

これだけで、エージェントによるコーディングが始まる。

**write requires approvalと表示された場合**:

恐らく、"write requires approval" と表示され、以下の選択肢が表示される。

```bash
 ❯ Yes, single permission
   Trust, always allow in this session
   No
```

- Yes, single permission: 今回だけ実行を許可する
- Trust, always allow in this session: 今回のセッションでは実行を許可する
  - 選択すると、どんなコマンドを許可するかの選択肢が表示される
- No: 実行を拒否する

最終的に完了した旨のコメントが来たら、作業完了です。

コメント例:

```txt
テトリスが完成しました！

アクセスURL: http://52.198.239.234:8080 (http://52.198.239.234:8080)

操作方法:

- ← → : 左右移動
- ↑ : 回転
- ↓ : 下に加速
- スペース : 一気に落とす（ハードドロップ）
- START ボタン: ゲーム開始/リスタート
```

**接続方法を教えてもらえなかったら**:

例では接続方法を教えてくれているが、教えてくれない事もある。
その場合は「ゲームへの接続方法を教えてください。」と聞けば教えてくれる。

### 2.1.3. 機能を追加する

作成されたものは本当に最低限のデザインであり、ゲームとしての完成度は高くない。
そこで、エージェントと対話的にアップデートしていく。

**背景に罫線を追加する**:

```txt
ゲーム画面の背景に、罫線を追加して下さい。
```

**だんだん難易度を上げる**:

```txt
30秒ごとに、少しずつ落下速度が上がるようにして下さい。
```

このように、AI コーディングエージェントと対話的にアプリを開発していく手法を **Vibe Coding(バイブコーディング)** と呼ぶ。
