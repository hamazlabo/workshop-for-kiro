# プロダクト概要 / ワークショップの目的

このリポジトリは、**Kiro CLI と spec 駆動開発（SDD）を体験するハンズオンワークショップ**の作業用リポジトリです。
参加者は code-server（ブラウザ版 VS Code）上でこのリポジトリを開き、2 つのアプリを順番に開発します。

## ワークショップで作る 2 つのアプリ

### 1 回目: テトリス（クイック開発 & 公開）

- **内容**: ブラウザで遊べる簡単なテトリスを作る。
- **狙い**: code-server 環境に慣れ、作ったものを**手早く公開する**流れを体験する。
- **進め方**: SDD は使わず、対話しながらサッと実装してそのまま公開する（vibe coding 寄り）。
- **公開**: Web サーバとして直接公開し、`http://<パブリックIP>:8080` で遊べるようにする。

### 2 回目: AI チャット（spec 駆動開発）

- **内容**: 簡単な AI チャットアプリを作る。
- **狙い**: **spec 駆動開発（sdd-plan → sdd-develop）の一連の流れ**を体験する。requirements → design → plan → task の 4 ドキュメントを作ってから実装する。
- **技術の要点**:
  - エージェントフレームワーク: **Strands Agents（Python）**
  - 会話履歴管理: **FileSessionManager**（会話状態をファイルに永続化）
  - UI: **Streamlit**（ブラウザ UI のチャット）
- **公開**: Streamlit として `http://<パブリックIP>:8501` で公開する。

## 共通の方針

- 完成したらブラウザでアクセスできる状態にし、**現在のパブリック IP を取得してアクセス URL をユーザに返す**（詳細は `AGENTS.md`）。
- ワークショップ規模に合わせ、過度に作り込まず「動いて公開できる」ことを優先する。
